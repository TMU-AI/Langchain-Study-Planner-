# New to GitHub? Start Here.

This guide assumes:
- You are intelligent
- You are motivated
- You may have never used GitHub before

No prior knowledge required.

---

## The Core Idea

GitHub lets many people work on the same project **without overwriting each other**.

Think of it like Google Docs, but:
- Much more powerful
- Much less forgiving
- Designed by engineers who enjoy precision

---

## Key Words You Must Understand

### Repository (Repo)
A project folder that lives on the internet.

### Branch
A **copy of the code** where you can safely experiment.

You will work on your own branch.  
You will *not* work on `main`.

### Commit
A saved checkpoint with a message explaining *what* you changed and *why*.

### Push
Uploading your commits to GitHub.

### Pull Request (PR)
A request to merge your work into `main`.

This is how collaboration happens.
This is normal.
This is expected.
